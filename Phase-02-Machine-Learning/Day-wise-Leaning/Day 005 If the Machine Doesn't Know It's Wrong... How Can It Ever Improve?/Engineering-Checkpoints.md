

# Engineering Checkpoint

## 1. What new mental model did I build today?

Today, I understood that learning is driven by feedback. Improvement doesn't come from repeating the same action—it comes from understanding the result of each attempt and using that information to make the next attempt better.

---

## 2. What misconception did I correct today?

Earlier, I thought practice alone leads to improvement. Today, I realized that practice without meaningful feedback can simply reinforce the same mistakes. Feedback is what turns practice into learning.

---

## 3. What question is still confusing me?

I understand that feedback helps improve performance, but I'm curious about how a machine uses that feedback to decide what changes it should make after each attempt.

---

## 4. If I had to teach today's lesson in 2 minutes, what would I say?

Imagine you're learning to play basketball.

You take a shot.

If nobody tells you whether it went in, whether it was too short, too long, or too far to the left, you'll probably keep shooting the same way every time.

Now imagine someone gives you feedback after every shot.

"This one was too strong."

"This one was a little to the left."

"This one was almost perfect."

With every attempt, you make a small improvement based on that feedback.

Over time, your shots become more accurate.

Learning works like this in many areas of life. We try something, observe the result, make adjustments, and try again. The cycle of **attempt → feedback → improvement** is what helps us get better.

The same idea will guide everything we study in Machine Learning. Before a machine can improve, it must first receive feedback about how well it performed.
