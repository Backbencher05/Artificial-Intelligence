# The Big Idea

Learning isn't just about knowing:

> "I was wrong."

It's about answering two questions:

1. **What should I change?**
2. **How much should I change it?**

Every learning system—human or machine—must solve these two problems.

---

# A Very Important Thought

Notice something.

The machine doesn't immediately jump to the perfect settings.

It improves **gradually**.

Why?

Because giant changes are risky.

Imagine you're learning to drive.

Every mistake doesn't require changing your entire driving style.

Usually...

A small steering correction is enough.

Machine Learning often works the same way.

---

# Engineering Perspective

Think about tuning a production system.

You wouldn't change:

* Database
* Cache
* Load balancer
* Programming language
* Cloud provider

...all at once.

Why?

Because then you wouldn't know which change helped or hurt.

Engineers prefer:

* Measure.
* Change one thing.
* Measure again.

ML algorithms also rely on controlled, incremental adjustments.
