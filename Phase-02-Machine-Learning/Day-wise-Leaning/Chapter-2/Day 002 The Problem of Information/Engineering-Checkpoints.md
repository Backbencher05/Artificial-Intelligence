
# Engineering Checkpoint

## 1. What new mental model did I build today?

I learned that collecting more data does not automatically lead to better decisions. The key is identifying the information that is actually relevant to the problem. Useful data helps make accurate predictions, while irrelevant data adds noise and complexity.

---

## 2. What misconception did I correct?

I used to think that more information is always better. Now I understand that irrelevant information can distract both humans and machine learning models, making them less effective.

---

## 3. What question is still confusing me?

How do machine learning engineers determine which pieces of information are useful when working with very large datasets containing hundreds or thousands of features?

---

## 4. If I had to teach today's lesson in 2 minutes, what would I say?

Before a machine can learn, it needs data that represents reality. However, not every piece of data is useful. The goal is to identify the information that has a meaningful relationship with the prediction task and ignore irrelevant details. This process helps models learn faster, make better predictions, and avoid being distracted by noise.
